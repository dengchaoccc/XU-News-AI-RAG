个性化新闻智能知识库项目

一、 功能要求
1.实现定时任务机制 ，通过 RSS、 网页抓取及智能代理工具获取新闻信息（需遵守网络爬虫规范与安全准则 ），借助 n8n 的定时触发节点与数据抓取节点完成自动化信息采集流程。
2.基于 Ollama 部署大模型，推荐采用 qwen3 模型，通过 n8n 的 HTTP 请求节点或自定义节点实现与 Ollama 服务的交互。
3.构建本地知识库系统 ，配置嵌入模型（推荐使用 all-Mini LM-L6-v2 ）、重排模型（推荐使用 ms-marco-Mini LM-L-6-v2 ）及大语言模型 ，利用 n8n 的节点串联能力实现模型调用、数据处理与流转。
4.通过 n8n 的 API 节点、数据库节点等将抓取的信息写入知识库系统 ，支持结构化数据（可通过 Excel 整理）与非结构化数据类型 ，实现数据自动化入库。
5.信息入库成功后 ，通过 n8n 的邮件发送节点自动发送邮件提示（标题及内容自定义） ，可配置邮件模板与触发条件。
6.提供用户登录功能 ，通过 n8n 集成身份认证相关节点或对接外部认证服务实现。
7.登录后支持知识库内容管理 ：查看数据列表（可按类型 / 时间筛选）、执行单条或批量删除操作、编辑元数据（如标签、来源） ，并支持通过页面上传多种类型数据至知识库 ；前端操作通过 API 与 n8n 工作流联动 ，实现数据管理流程自动化。
8.登录后提供语义查询功能 ：基于用户提问优先检索知识库内容 ，通过 n8n 串联向量数据库查询节点、相似度排序节点 ，结果按相似度排序返回。
9.若知识库未匹配到相关数据 ，通过 n8n 自动触发联网查询工作流（如调用百度搜索API 节点 ），获取前 3 条结果后 ，联动大模型推理节点处理并输出结果。
10.登录后提供知识库数据聚类分析报告 ，展示关键词 Top10 分布 ；通过 n8n 的数

据统计节点、聚类分析节点处理知识库数据 ，生成分析结果并同步至前端展示。

二、 技术要求
1.技术栈选型 ：前端采用 React 框架 ，后端采用 SpringBoot 框架 ，关系型数据库选用 MySQL ，向量数据库采用 FAISS ；核心流程自动化依赖 n8n 工具 ，通过其节点生态打通各系统与工具。
2.数据存储设计 ：元数据（如数据 ID、数据类型等）存储于关系型数据库 ，向量数据单独存储于 FAISS 向量数据库 ，实现数据分类管理与高效检索 ；n8n 通过数据库连接节点（如 MySQL 节点、 自定义 FAISS 交互节点 ）实现数据的读写与同步。
3.工作流集成 ：核心业务逻辑（数据采集、入库、查询、分析、通知等）通过 n8n 设计可视化工作流实现 ，无需复杂代码开发即可完成异构系统间的联动。
4.大模型调用：通过 n8n 的 HTTP 节点、自定义模型调用节点等标准化方式实现与大模型服务的交互 ，确保服务调用的规范性与可扩展性 ，支持工作流内灵活调整调用逻辑。
5.身份认证 ：登录功能优先采用 Spring Security 框架或 JWT 技术方案 ，通过 n8n的认证节点、API 请求拦截节点等实现身份验证结果与工作流的联动 ，保障用户身份验证的安全性与可靠性。
6.私有化部署与合规 ：n8n 支持私有化部署 ，所有数据处理均在本地服务器完成 ，不流经第三方公有云 ，确保数据主权 ，完全符合欧盟 GDPR 及德国企业对数据隐私的合规要求。
